# 服务命名/目录命名规范

<br/>

1. 我们将服务区分为两种，一种为 basics 作为中间件基础服务，另一种为 service 作为业务服务
   
   <br/>
2. 服务的名称会命名为 maozi-cloud-[basics|service]-[服务名称]，如：
   ```
   # basics 类型
   maozi-cloud-basics-gateway
   
   # service 类型
   maozi-cloud-service-user
   ```

<br/>

3. 这两种的代码目录也不一致，basics类型的只有一层根目录，service类型的会有两层目录，如：
   ```
   # basics 类型
   maozi-cloud-basics-[服务名]                            (核心实现代码)
   
     
   # service 类型
   maozi-cloud-[服务名]
     maozi-cloud-[服务名]-dto                             (提供给外部的传输模型)
     maozi-cloud-[服务名]-enum                            (提供给外部的枚举模型)
     maozi-cloud-[服务名]-vo                              (提供给外部的视图模型)
     maozi-cloud-service-rest-api-[服务名]                (提供给外部的HTTP接口)
     maozi-cloud-service-rpc-api-[服务名]                 (提供给外部的RPC接口)
     maozi-cloud-service-[服务名]                         (核心实现代码)
   ```

<br/>

4. service类型包路径说明
   ```
   maozi-cloud-xxx-dto
     com.maozi.xxx.[子模块].dto.v[版本 任意数值].[服务于哪个端的如：ios|android|global|platform].[dto|param]
   
   maozi-cloud-xxx-enum
     com.maozi.xxx.[子模块].enums
     
   maozi-cloud-xxx-vo
     com.maozi.xxx.[子模块].vo.v[版本 任意数值].[服务于哪个端的如：ios|android|global|platform]
     
   maozi-cloud-service-rest-api-xxx
     com.maozi.xxx.[子模块].api.rest.v[版本 任意数值].[服务于哪个端的如：ios|android|global|platform]
     
   maozi-cloud-service-rpc-api-xxx
     com.maozi.xxx.[子模块].api.rpc.v[版本 任意数值]
   ```

<br/>

5. service核心代码包路径
   ```
   maozi-cloud-servicce-[服务名]
     com.maozi.                                                                        (启动入口)
       [服务名].[子模块].                                                              
         api.impl.
           rpc.
           rest.
             v[版本 任意数值].[服务于哪个端的如：ios|android|global|platform]          (接口实现)
         domain                                                                        (领域模型映射数据库)
         mapper                                                                        (领域模型)
         properties                                                                    (参数配置)
         config                                                                        (配置)
         utils                                                                         (工具)
   ```

<br/>

6. service核心代码类名称
   ```
   # DTO
   # 入参
   XxxParam.java
   
   # DTO
   # 响应
   Xxx.java
   
   # Vo
   XxxVo.java
   
   # Rest Api
   RestXxxServiceV[版本 任意数值].java
   
   # Rpc Api
   RpcXxxServiceV[版本 任意数值].java
   
   # Run
   [服务模]Application.java
   
   # Do
   [表名]Do.java
   
   # Mapper
   [表名]Mapper.java
   
   # Properties
   XxxProperties.java
   
   # Utils
   XxxUtils.java
   
   # Config
   XxxConfig.java
   
   # Service Api
   XxxService.java
   
   # Service Api Impl
   XxxServiceImpl.java
   
   # Rest Service Api Impl
   RestXxxServiceImplV[版本 任意数值].java
   
   # Rpc Service Api Impl
   RpcXxxServiceImplV[版本 任意数值].java
   ```
